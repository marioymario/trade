Answer:
The RAG (Retrieval-Augmented Generation) system is deployed and used through the `rag.sh` script located in the `rag` directory. The script manages the Docker container using the `docker-compose.rag.yml` configuration file.

1. **Deployment**: When you run `./rag.sh` without any arguments, it ensures that the RAG service is up and running using Docker Compose and then executes `python query.py` to interact with the system.

2. **Indexing/Reindexing**: If you run `./rag.sh index` or `./rag.sh reindex`, it ensures the RAG service is up and runs `python ingest_repo.py` to index the repository.

3. **Querying**: For other commands or no arguments, it runs `python query.py` with the provided arguments or without any if none are given.
